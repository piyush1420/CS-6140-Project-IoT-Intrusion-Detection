# Lightweight Intrusion Detection for IoT Networks

**CS 6140 — Machine Learning**
**Team:** Mahip Parekh & Piyush Daga

## Project Overview

Feature-efficient intrusion detection for IoT networks using the RT-IoT2022 dataset. We compare feature selection methods (Random Forest importance, PCA) and class imbalance strategies (SMOTE, cost-sensitive learning) across four classifiers: Logistic Regression, SVM, Random Forest, and DNN (PyTorch).

We run **60 experiments** (4 models x 5 feature sets x 3 imbalance strategies) and evaluate using **macro-averaged F1** as the primary metric.

**Best result:** Random Forest on all 83 features with no imbalance handling achieves **Macro F1 = 0.9763**.

## Dataset

**RT-IoT2022** from the UCI Machine Learning Repository.

- **Source:** https://doi.org/10.24432/C5P338
- **Samples:** 123,117 network flows
- **Features:** 83 (81 numerical + 2 categorical: `proto`, `service`)
- **Classes:** 12 attack/device types (highly imbalanced — dominant class is 76.9% of data)

The dataset is not included in this repository due to size. See setup instructions below.

## Setup & Replication

### Prerequisites

- Python 3.9+
- pip

### Step 1: Clone the repository

```bash
git clone https://github.com/piyush1420/CS-6140-Project-IoT-Intrusion-Detection.git
cd CS-6140-Project-IoT-Intrusion-Detection
```

### Step 2: Install dependencies

```bash
pip install -r requirements.txt
```

### Step 3: Download the dataset

Download the RT-IoT2022 dataset from the UCI ML Repository:

https://doi.org/10.24432/C5P338

Place the CSV file at `data/RT_IOT2022` (no file extension). Create the `data/` directory if it doesn't exist:

```bash
mkdir -p data
# Move or copy the downloaded CSV into data/RT_IOT2022
```

### Step 4: Run notebooks in order

Run the notebooks in the following order. Each phase depends on outputs from the previous phase.

**Phase 1 — Data Exploration & Preprocessing:**
```
notebooks/piyush/01_data_exploration.ipynb
```
Loads the raw dataset, encodes features, creates the stratified train/test split, and saves preprocessed data to `data/processed/`.

**Phase 2 — Feature Selection:**
```
notebooks/piyush/02_feature_selection.ipynb
```
Computes RF importance rankings and PCA. Saves 5 feature subsets (full-83, top-30, top-20, top-10, pca-26) to `data/processed/`.

**Phase 3 — Model Training (these 4 notebooks can be run in any order):**
```
notebooks/mahip/01_imbalance_and_dnn.ipynb        # DNN (15 experiments)
notebooks/mahip/02_logistic_regression.ipynb       # Logistic Regression (15 experiments)
notebooks/mahip/03_random_forest.ipynb             # Random Forest (15 experiments)
notebooks/piyush/03_svm_training.ipynb             # SVM (15 experiments)
```
Each notebook trains its model on all 5 feature sets with 3 imbalance strategies (baseline, SMOTE, cost-sensitive). Saves models to `models/` and results to `results/`.

**Phase 4 — Evaluation & Visualization:**
```
notebooks/piyush/04_evaluation.ipynb
```
Loads all 60 experiment results, generates comparison figures, and creates the master results table.

## Experimental Design

### Models
| Model | Key Hyperparameters |
|---|---|
| Logistic Regression | max_iter=1000, L2 regularization |
| SVM | RBF kernel, C=1.0, gamma=scale (SGD linear SVM for SMOTE variant) |
| Random Forest | n_estimators=100, no max depth limit |
| DNN (PyTorch) | 4 layers (128→64→32→12), BatchNorm, Dropout=0.3, Adam lr=0.001 |

### Feature Sets
| Feature Set | Description |
|---|---|
| full-83 | All 83 features |
| top-30 | Top 30 by RF importance (75% cumulative importance) |
| top-20 | Top 20 by RF importance (60% cumulative importance) |
| top-10 | Top 10 by RF importance (36.7% cumulative importance) |
| pca | PCA with 26 components (95% variance retained) |

### Imbalance Strategies
| Strategy | Description |
|---|---|
| Baseline | No imbalance handling |
| SMOTE | Oversample minority classes in training data (98K → 909K samples) |
| Cost-sensitive | Balanced class weights (inverse class frequency) |

## Results Summary

### Best Configuration Per Model

| Model | Best Config | Macro F1 | Accuracy |
|---|---|---|---|
| **Random Forest** | full-83, baseline | **0.9763** | 0.9985 |
| DNN | top-20, baseline | 0.9515 | 0.9946 |
| SVM | full-83, baseline | 0.9402 | 0.9940 |
| Logistic Regression | full-83, baseline | 0.9179 | 0.9911 |

### Key Findings

1. **Random Forest dominates** across all feature sets and imbalance strategies.
2. **Baseline outperforms SMOTE and cost-sensitive** for strong models (RF, SVM). Feature selection matters more than imbalance handling.
3. **DNN improves with fewer features** — Macro F1 jumps from 0.79 (83 features) to 0.95 (20 features), showing that feature reduction acts as regularization.
4. **20 features achieve 98.3% of full performance** for RF (76% dimensionality reduction with only 1.7% F1 drop).
5. **Rare attacks are well-detected** — even classes with only 6-7 test samples achieve >87% F1 with the best RF model.

Full results are in `results/master_results.csv`. All figures are in `figures/`.
